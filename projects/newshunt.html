Brief Summary

News-Hunt is a single-page application that pull trending posts from a series of external news sources using API REST calls.

The front-end was built using the CSS framework Materalize and JavaScript framework Vue.js. While the back-end was built using the Ruby on Rails framework.

News-Hunt allows its users to discover trending posts from Hacker News, Reddit, Product Hunt, and GitHub Trending. For an optimized experience, a user can sign up and start bookmarking their favorite posts to read later.

Technologies

Vue.js

An open-source JavaScript framework for building user interfaces.

Ruby on Rails

A server-side web application framework written in Ruby.

Overview



Starting with the navigation bar, you can see which site APIs we decided to pull from. With v-on:click handeling, our site would send a GET request to the targeted API, reformat the information, and load the retrieved Posts as cards within a load time of 560ms on average.

Our ALL tab started with a load time averaging 4.5 seconds, before we were able to target a specific API that was causing the significant delay, which we were able to refactor and bring down to an average of 2 seconds.

Our goal is to optimize this rendering even further, by updating our back-end logic to have our server store the results of periodic GET requests in the background so that instead of submittting four GET requests to multiple APIs at the time of the user's click handeling, our front-end would simply make one request to our server, thus reducing our loadtime.



We built in some very simple login/signup functionality, where a user can create an account by providing only an email address and a desired password.

Once logged in, the user then has the ability to add posts to their bookmarks for future reading by clicking the gray 'plus' icons found on each Post card.

This feature was simple to design in the back-end, where there was a database table for Users and Bookmarks with a has-many / belongs-to relationship.

The challenging part of this implementation was connecting the front-end module to the back-end logging in and authenticating user functionality, which was then solved through plenty of trial-and-error programming.
