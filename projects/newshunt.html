Brief Summary

News-Hunt is a single-page application that pulls trending posts from a series of external news sources using API REST calls, developed by myself and programming partner Robert Rosenberg.

Technologies

Vue.js

An open-source JavaScript framework for building user interfaces.

Ruby on Rails

A server-side web application framework written in Ruby.

Overview



Starting with the navigation bar, you can see which site APIs we decided to pull from. With on-click handeling, our site would send a GET request to the targeted API, reformat the retrieved data, and render the Posts as cards within an average load time of 560ms. Of course though, some API requests took longer than others, which would affect our home page load time, which rendered ALL posts.


With our first draft, our ALL request had an average load time of 4.5 seconds, which we understood was terrible for any user experience. We therefore fixed this issue by targeting the specific API that was causing the significant delay, refactoring how we were sending the request and iterating through the information, and thus brought down the load time to a new average of 2 seconds.

Our goal however is to optimize this rendering even further. By updating our backend logic to have our server store the results of periodic GET requests in the background, we could transition away from submitting multiple GET requests to multiple APIs at the time of the user's click handeling. Instead, our frontend would simply make one request to our server, thus reducing the average loadtime.

The features we added to the Post cards were hyperlinks to the pages where the post or comment thread originated. Plus we included with each card the source of the post, how many points, if any, the post had, and who the author was. However, we do not load the full content internally on our platform; News-Hunt simply links our users directly to the original sources.



To enhance our user experience, we included a signup/login feature that would only ask for an email address, password and password confirmation with the mentality that the less forms a user would have to fill out, the more likely they'll sign up.

Once logged in, the user would then have the ability to add posts to their bookmarks for future reading by clicking the gray 'plus' icons found on each Post card and clicking on the Boomkarks tab.

This feature was simple to design in the backend, with a database table for Users and Bookmarks with a has-many / belongs-to relationship.

The challenging part of this implementation was setting up the login/signup functionality with user authentication. After reading a fair amount of articles on authentication and designing a login module, we were able to integrate it into our code to keep the user experience fluid and backend functioning properly.
