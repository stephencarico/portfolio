Brief Summary

TalentStack is a web application designed to connect entrepreneurs with similar ideas and interests. Through database modeling the application allows users to sign up and post their ideas to receive feedback, advice and potentially find others interested in helping them pursue their vision.

Technologies

Vue.js

An open-source JavaScript framework for building user interfaces.

Ruby on Rails

A server-side web application framework written in Ruby.

Overview



Features that I'd like to highlight include the category tags, signup/login functionality, and the DOM manipulation dependent on if a user is logged in.

At the top of the Home page, there is a list of tags that when clicked on will display posts that have that tag association. Each post is allowed to have multiple tags and therefore can be found in many of the nine predetermined tags. Users are not allowed to create tags for site simplicity.

Here is the new post page, that a user fills in to create a new post. Each new idea must have a title, a pitch and an overview, otherwise an error will be triggered prompting the user to fix the appropriate field.

The posts edit page looks the same. The challenging part with the update functionality was the many-to-many relationship that posts had with tags. When working with the front-end, I wanted the user to only be able to choose between the nine prompted tags, so therefore radio buttons seemed like the best option.


When viewing posts, if you are logged in, an Edit button will appear next to the title of the post. Beneath the post title, breadcrumbs of information are embedded, including: post date, name of author, and tags the post is associated with.

At the bottom of every post there is a comment section where other users can leave comments. Users also have the ability to delete their comments if the comment belongs to them.



Ever user has a profile page that includes a card that includes their name, tagline, and links to their other social media platforms. Next to their profile card, a list of their authored posts is displayed for easy access.

For easy user navigation, the Create Post button can be seen in both the navigation bar and on the user's profile page below their profile card.

Where is the project now? It is still in development. My plans are to seperate the back-end from the front-end so that everything isn't running in the same place. I may end up using React in stead of Vue.js when rebuilding the front-end from the ground up due to it's strength in scalability and my interest in learning more about this framework.

I do believe in this idea, primarily because this is the day and age where many people are trying to start their own business, but simply don't have the right tools, guidance or experience. With this site's main feature of idea security, I hope to create this pre-kickstarter phase site to help aspiring entrepreneurs like myself achieve their dreams.
